# Support

Before requesting help:

1. Read the [documentation index](docs/README.md).
2. Run `gitnodes doctor /path/to/brain`.
3. Check [known limitations](docs/OPERATOR_NOTES.md#what-it-does-not-promise).
4. Search existing GitHub issues.

Use a bug report for reproducible product failures and a discussion or feature
request for usage questions and product ideas. Include `gitnodes --version`,
the operating system, the execution mode (`preview`, `mcp`, local `serve`, or
hosted), and redacted doctor output.

Never post access tokens, OAuth secrets, webhook secrets, private keys, session
keys, or private repository content.
